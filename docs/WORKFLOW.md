# SAPHO 基因分析 —— 完整工作流说明（范本模式）

本文档定义了一套 **可复用的基因分析工作流范本**。它不只适用于SAPHO，也适用于任何基因-疾病关联分析。

## 工作流架构

```
┌─────────────────────────────────────────────────────┐
│            层级1：知识库（静态信息源）               │
│  docs/SAPHO_knowledge_base.md                      │
│  - 疾病概述、已知关联基因、变异数据、通路信息       │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│            层级2：数据采集（动态信息源）             │
│  scripts/sapho_analysis_pipeline.py                │
│  - ClinVar API → 已知变异数据                       │
│  - OpenTargets API → 靶点-疾病关联                  │
│  - PubMed API → 最新文献                            │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│            层级3：AI分析（大模型推理）               │
│  - 变异致病性评估                                   │
│  - 蛋白质结构影响预测                               │
│  - 文献证据综合                                     │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│            层级4：输出（结构化成果）                 │
│  output/{gene}_report.md                           │
│  - 基本信息、变异清单、证据等级、下一步建议         │
└─────────────────────────────────────────────────────┘
```

## 范本模式（可复用模板）

以下模式可套用到任何基因-疾病关联分析课题：

### 模板A：单基因深度分析
- **输入**: 基因名
- **处理**: 知识库匹配 → API查询(ClinVar/PubMed) → 结构化输出
- **输出**: 单基因Markdown报告
- **适应症**: 验证某个基因与疾病的关联

### 模板B：多基因比较分析
- **输入**: 基因列表
- **处理**: 批量分析 → 一致性比较 → 综合评分
- **输出**: 多基因比较表格 + 独立详细报告
- **适应症**: 队列研究中筛选候选基因

### 模板C：竞赛/悬赏机会监控
- **输入**: 平台列表(Kaggle/Tianchi/基金)
- **处理**: 定期抓取 → 关键词匹配(SAPHO/罕见病/基因) → 通知
- **输出**: 机会汇总表 + 优先级排序
- **适应症**: 追踪可参与的赛事和基金申请窗口

### 模板D：文献自动综述
- **输入**: 基因列表 + PubMed API
- **处理**: 批量检索摘要 → LLM总结 → 证据等级分类
- **输出**: 文献综述Markdown
- **适应症**: 快速了解某个课题的研究现状

## 扩展指南

### 增加新基因
1. 编辑 `scripts/sapho_analysis_pipeline.py` 中的 `SAPHO_KNOWLEDGE_BASE` 字典
2. 格式参考已有基因条目（需包含gene_name, locus, known_variants等字段）
3. 更新 `docs/SAPHO_knowledge_base.md` 中的基因表格

### 增加外部数据源
1. 在 `scripts/sapho_analysis_pipeline.py` 中新增 `query_XXX()` 函数
2. 在 `generate_report()` 中调用该函数
3. 推荐数据源：Ensembl REST API、UniProt API、DisGeNET API

### 切换到其他疾病
只需替换 `SAPHO_KNOWLEDGE_BASE` 为目标疾病的知识库，工作流代码无需修改。

## 常见问题

**Q: API查询失败怎么办？**
- 检查网络连接，确认能访问 ncbi.nlm.nih.gov
- 某些API有访问频率限制，添加时间间隔

**Q: 如何验证分析结果？**
- 交叉核对ClinVar的致病性判定与已发表文献
- 对于未知变异，使用SIFT/PolyPhen等预测工具

**Q: 这个工作流能直接用于竞赛吗？**
- 可提供分析和报告框架，但竞赛需要实际数据和参赛要求
- 建议先用本工作流做前期调研，再针对具体竞赛调整
